# 前端 Vue 代码风格规范

## 核心原则

前端代码应该先像一份可读的界面结构说明，再像一份具体实现。

每个文件都应该让阅读者快速看明白：

- 这个文件主要负责什么。
- 当前界面有哪些按钮、面板、拖动条、菜单、开关、标签页、输入框等控件。
- 每个控件分别负责什么交互。
- 哪些 API 调用只属于某个具体控件。
- 哪些逻辑是通用能力，应该放到代码底部统一管理。

优先保证结构清晰，不追求把代码压得很短。避免代码横向写成很长一串。

## 文件开头注释

每个 Vue、JavaScript、TypeScript、CSS 前端源码文件，开头都必须有中文注释框，说明这个文件的主要职责。

除非用户明确要求，或者文件职责发生了真实变化，否则不要改动文件级注释。

普通脚本或样式文件使用这个格式：

```js
/*
|--------------------------------------------------------------------------
| 文件模块名称
|--------------------------------------------------------------------------
| 用中文说明这个代码文件主要负责什么、服务哪个界面或功能。
|--------------------------------------------------------------------------
*/
```

`.vue` 文件开头可以使用这个格式：

```vue
<!--
|--------------------------------------------------------------------------
| 文件模块名称
|--------------------------------------------------------------------------
| 用中文说明这个代码文件主要负责什么、服务哪个界面或功能。
|--------------------------------------------------------------------------
-->
```

## Vue 文件代码顺序

Vue 文件建议按这个顺序组织：

1. 文件开头注释。
2. `import` 引入。
3. 交互控件清单。
4. 界面状态。
5. 控件专属样式。
6. 控件专属事件和 API。
7. 不属于单个控件的功能逻辑。
8. 共用工具函数放到底部。
9. `<template>` 按可见 UI 模块分组。

这样阅读者可以先理解界面有哪些控件，再理解每个控件背后的实现。

## 交互控件清单

在 `<script setup>` 的顶部，用中文注释框列出当前文件包含的主要控件。

控件包括但不限于：

- 按钮
- 面板
- 拖动条
- 输入框
- 开关
- 标签页
- 菜单
- 拖拽区域
- 进度条
- 状态标识

示例：

```js
/*
|--------------------------------------------------------------------------
| 交互控件清单
|--------------------------------------------------------------------------
| 选择资源按钮：打开或模拟选择资源文件。
| 开始转码按钮：触发当前资源的转码流程。
| 状态面板：展示当前文件和后端处理状态。
|--------------------------------------------------------------------------
*/
```

## 模块注释框

重要代码模块前必须使用中文注释框说明职责。

适合添加模块注释的地方：

- 一组相关状态。
- 某个按钮的样式和点击事件。
- 某个面板展示的数据。
- 某个拖动条或输入框的数值和校验。
- 某个功能的后端 API 调用。
- 一组可复用工具函数。

示例：

```js
/*
|--------------------------------------------------------------------------
| 选择资源按钮
|--------------------------------------------------------------------------
| 负责写入当前资源文件，并向后端查询该资源的初始处理状态。
|--------------------------------------------------------------------------
*/
const selectAssetButtonClass = [
  "rounded-md",
  "bg-cyan-400",
  "px-4",
  "py-2"
];

async function handleSelectAsset() {
  selectedFile.value = "sample-image.png";
  status.value       = await requestAssetStatus(selectedFile.value);
}
```

## 代码对齐和可读性

避免代码在视觉上横向拉得很长。

必须遵守：

- 很长的 Tailwind class 不要写成一行，拆成数组。
- 一个复杂控件的 Tailwind class 尽量一行一个。
- 相邻的简单赋值可以适当对齐，提高扫描速度。
- 交互元素的属性和事件一行写一个。
- 事件函数命名要清楚，例如 `handleSelectAsset`、`handleStartTranscode`、`requestAssetStatus`。

推荐：

```vue
<button
  :class = "selectAssetButtonClass"
  type   = "button"
  @click = "handleSelectAsset"
>
  选择资源
</button>
```

避免：

```vue
<button class="rounded-md bg-cyan-400 px-4 py-2 text-sm font-semibold" type="button" @click="handleSelectAsset">
  选择资源
</button>
```

## API 归属规则

如果某个 API 只服务一个具体控件，就把 API 包装函数放在该控件模块附近。

示例：

```js
/*
|--------------------------------------------------------------------------
| 选择资源按钮 API
|--------------------------------------------------------------------------
| 仅服务选择资源按钮，用于查询当前资源在后端的处理状态。
|--------------------------------------------------------------------------
*/
async function requestSelectedAssetStatus(path) {
  return window.go.main.App.TranscodeStatus(path);
}
```

如果某个 API 被多个控件或多个模块复用，就放到底部的共用 API 模块中。

示例：

```js
/*
|--------------------------------------------------------------------------
| 共用后端 API
|--------------------------------------------------------------------------
| 放置多个控件都会使用的后端调用，避免散落在不同交互模块里。
|--------------------------------------------------------------------------
*/
async function requestBackendHealth() {
  return window.go.main.App.Health();
}
```

## 模板结构

在 `<template>` 中，按照可见 UI 模块分组，并在每组前面加中文注释框。

使用这个结构：

```vue
<!--
|--------------------------------------------------------------------------
| 操作按钮模块
|--------------------------------------------------------------------------
| 放置资源选择、开始转码等主要操作按钮。
|--------------------------------------------------------------------------
-->
<div class="flex flex-wrap gap-3">
  ...
</div>
```

按钮、面板、拖动条等控件应遵守：

- 属性竖向排列。
- 可见文字保持简洁。
- 注释说明控件职责，不要逐行解释语法。
- 控件专属的状态、样式、事件、API 尽量放在同一个代码区域。

## 底部放置规则

这些内容应该放在 `<script setup>` 底部：

- 通用格式化函数。
- 纯工具函数。
- 共用校验函数。
- 共用后端 API 包装。
- 不属于某个可见控件的常量。

不要把通用工具函数插在某个按钮和该按钮事件之间，除非这个工具函数只属于这个按钮。

## 验证要求

修改 Vue 前端代码后：

1. 运行前端构建。
2. 如果改到了 Go 或 Wails 绑定，进入 `backend_go` 运行 Go 编译。
3. 如果当前目录是 git 仓库，按项目要求处理 pull 和 push。
